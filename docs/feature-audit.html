01 Executive Summary

Daily Meds is a full-featured meditation and wellness web application built on Next.js, Supabase, and Stripe. The codebase contains 46 distinct features spanning session playback, subscription management, live video rooms, social community, affiliate marketing, and a complete admin CMS. Of these, 40 are fully complete and 4 are partially implemented. The app integrates 8 external services, manages 28 database tables, and runs 6 scheduled cron jobs for automated engagement. The product is currently behind a waitlist gate—all features are built and functional but not yet publicly accessible. The primary gaps are cosmetic (placeholder content on 2 pages) and a localStorage/Supabase dual-implementation issue affecting watchlist and downloads.

03 Detailed Feature Descriptions

Expand each section for implementation details and dependencies.

Core Product 4/4 Complete
  • Session Library — 31 sessions across 11 mood categories (Calm, Focus, Energy, Sleep, Gratitude, Healing, Confidence, Joy, Resilience, Balance, Clarity). Supports filtering by mood, duration, and media type. Search across session titles and descriptions. Falls back to mock data in sessions-data.ts when Supabase has no data.
  • Session Player — Cinematic fullscreen experience. Handles audio and video playback with subscription-tier gating (free users see preview only). Resume position saved to Supabase via user_progress table. Picture-in-Picture support. Share modal generates OG images. Playlist modal for sequential playback.
  • Audio Player (Global) — Persistent mini-player fixed to bottom of viewport. Saves playback position every 10 seconds to Supabase. Automatically marks sessions as completed. Continues playing across page navigations.
  • Mood Gradients — 11 distinct CSS gradient definitions used consistently across session cards, player backgrounds, OG images, and share cards. Each mood has a primary and secondary color pair.
Subscriptions & Payments 5/5 Complete
  • Pricing Page — 6 subscription tiers:
    • Free (Observer) — Limited access
    • Monthly Listener — £9.99/month
    • Annual Listener — £99.99/year
    • Seeker — £19.99/month
    • Dedicated — £199/year
    • Master (Lifetime) — £299.99 one-time
    Includes FAQ section and plan comparison table. Trial banner promotes the £1 setup + 14-day trial offer.
  • Stripe Checkout — Server-side session creation for all plan types. Trial combo creates a £1 setup fee with 14-day trial followed by £9.99/mo recurring. Handles both new and returning customers.
  • Stripe Webhook — Processes 5 event types: checkout.session.completed, customer.subscription.updated, customer.subscription.deleted, invoice.payment_succeeded, invoice.payment_failed. Triggers 7 distinct email types via Resend. Auto-creates Supabase auth accounts for guest checkouts (users who pay before signing up).
  • Upgrade to Lifetime — One-click upgrade path for existing subscribers. Uses saved payment method on file. Cancels existing subscription and creates lifetime record atomically.
  • Customer Portal — Stripe's hosted billing portal. Users can update payment method, view invoices, cancel subscription, and resubscribe.
User Accounts 3/3 Complete
  • Auth — Four auth methods: email/password signup, Google OAuth, magic link (passwordless email), and PKCE code exchange for secure token handling. Password reset flow with email confirmation. Session persistence via Supabase Auth cookies.
  • Profile — Avatar upload to Cloudflare R2. All profile fields are inline-editable (click to edit, save on blur). Subscription tier display with management links. Community privacy settings (visibility, activity sharing). Full account deletion with cascade cleanup.
  • Stats / Streaks — Tracks total sessions completed, total minutes meditated, current streak (consecutive days), and longest streak. Weekly bar chart visualization. Top moods breakdown showing most-used categories. All data from user_sessions table.
Live Sessions 2/3 Complete, 1 Partial
  • Live Session Page — Displays currently live sessions with "LIVE" badge and upcoming sessions with countdown timers. Three event types: Audio Hugs (group listening), Alchemy Rewire (guided transformation), Expert Series (guest teachers).
  • Daily.co Rooms — Admin interface to create and manage Daily.co video rooms. Token generation requires access_level >= 2 (subscriber). Host capabilities include muting participants and removing disruptive users. Room lifecycle managed through live_sessions table.
  • Live Room Page — Daily.co iframe integration works for video/audio. Emoji reaction overlay works (floating emojis on click). However, the chat panel uses hardcoded mock messages—not connected to any real-time messaging system.
Group Features 3/3 Complete
  • Group Meditation Rooms — Any user can create a scheduled group session. Generates a unique invite code. Built-in meditation timer with configurable duration. Participant sync via Supabase Realtime (see who joined, who is meditating). Messages stored in group_session_messages.
  • Crews — Private friend groups. Create a crew, share the invite code, and friends join. Activity feed shows completions and streaks from members. Privacy-first: the feed never reveals which specific session a member completed, only that they completed one.
  • Brand Crews — B2B extension of crews. Brands apply through a public form. Admin reviews and approves/rejects. Approved brands get a branded community page with custom theming. 5 brand types supported (Wellness, Fitness, Corporate, Education, Healthcare).
Community 3/3 Complete
  • Community Feed — Automated social posts generated when users complete sessions, hit streak milestones, or achieve platform milestones. 6 emoji reactions per post. Leaderboard showing top meditators by sessions, minutes, and streaks.
  • Presence System — Real-time display of "who is meditating right now" powered by Supabase Realtime subscriptions. Floating emoji animations appear between active users to create a sense of shared experience.
  • Notifications — In-app notification bell with unread count badge. Dropdown panel shows recent notifications. Live updates via Supabase Realtime—new notifications appear instantly without refresh.
Content 3/3 Complete
  • Podcasts — Full podcast library with episode browsing. Admin CMS supports all podcast fields including guest name, guest bio, transcript, timestamped highlights, and episode artwork.
  • Free Content — Dedicated landing page showing all sessions where is_free = true. Server-rendered for SEO. Designed as a top-of-funnel conversion page.
  • Video Player — Vimeo OEmbed integration with branded playback controls. Graceful fallback when a session has no video (shows audio-only mode). Respects subscription tier access.
Affiliate System 4/4 Complete
  • Affiliate Program — Two tiers: Standard (auto-enrolled on signup, 10% commission) and Artist (application required, 20% commission). Full marketing page explaining the program and benefits.
  • Click Tracking — Detects ?ref=CODE URL parameter. Stores affiliate code in localStorage for attribution. Atomic click increment via Supabase RPC function (prevents race conditions).
  • Affiliate Dashboard — "Partner Pulse" branded dashboard showing total clicks, conversions, earnings, and payout history. Copyable referral link. Visual tier progress indicator. Payouts table with status badges.
  • Admin Affiliate Overview — Admin view of all affiliates with sortable columns for earnings, clicks, and conversions. Progress bars for conversion rates. Export capability.
Admin Panel 8/8 Complete
  • Admin Dashboard — Overview metrics: total users, active subscribers, revenue (MRR), subscription tier breakdown (pie chart), and recent signups list.
  • Session CMS — Full CRUD for meditation sessions. Drag-and-drop audio upload to Cloudflare R2. Vimeo and YouTube video preview embedding. Draft/published toggle. All 11 mood categories, duration, and media type fields.
  • User Management — Paginated user list with search by name/email. Filter by subscription tier. View individual user details and activity.
  • Live Management — Schedule upcoming live sessions with date/time picker. Create Daily.co rooms on demand. "Go Live" button to start streaming. Bulk email blast to notify subscribers.
  • Review Management — Queue of user-submitted reviews. Approve to display on testimonials page, or reject with optional reason.
  • OG / Social Settings — Edit OpenGraph title, description, and image per page. Preview how links will appear when shared on social media.
  • Podcast CMS — Full episode management with all metadata fields. Supports transcript upload, highlight timestamps, and guest information.
  • Brand Crew Admin — Queue of brand applications. Approve to auto-create branded crew page and send welcome email. Reject with feedback.
Other Features 10/13 Complete, 3 Partial
  • Challenges — 4 challenge types: complete N sessions, maintain N-day streak, meditate N minutes, explore N mood categories. Users can create, join, and track progress. Completion triggers community feed post.
  • Watchlist Partial — Supabase watchlist table exists and the watchlist page reads from it. But the session player's heart/save toggle writes to localStorage instead. Dual implementation needs reconciliation.
  • Downloads Partial — Same dual-implementation pattern as watchlist. The downloads page reads from Supabase, the library writes to localStorage. No actual audio file download capability exists yet.
  • Playlists — Full CRUD. Drag-and-drop reorder of sessions within a playlist. Stored as text[] array of session IDs in Supabase. Playback integration with the global audio player.
  • Breathing Timer — Standalone breathing exercise tool with 5 preset durations. Animated breathing circle. "Trending sessions" cards displayed alongside for upsell.
  • Brand Partnerships — B2B pitch page describing the partnership program. Application form collects company details, brand type, and goals.
  • Festivals Page Partial — Page structure is complete but 3 of 4 team member entries contain placeholder text: [Role] and [Description].
  • Shop — Shopify Buy Button SDK integrated. Product grid layout ready. Awaiting Shopify store configuration and product creation to go live.
  • PWA — Full Progressive Web App: manifest.json, service worker with caching strategy, offline fallback page, and app shortcuts for quick-launch meditation.
  • Waitlist / Early Access — Current public homepage. Email submission upserts to Supabase early_access_signups table, adds to Resend audience, and triggers welcome email notification.
  • OG Image Generation — Dynamic server-side generation of 1200×630 OG images and 1080×1080 share cards. Uses mood gradients as backgrounds with session title and branding overlay.
  • Email Templates — 17 React Email templates: welcome, trial started, trial reminder (day 5), subscription confirmed, payment failed, cancellation, lifetime upgrade, weekly recap, monthly recap, streak reminder, inactivity nudge, live session reminder, affiliate welcome, affiliate earnings, brand crew approved, brand crew rejected, early access confirmation.
  • R2 File Upload — Two-step upload: server generates presigned URL, client uploads directly to Cloudflare R2. Admin-only routes. Supports audio (mp3, wav, m4a) and image (jpg, png, webp) file types.

08 Known Issues

9 items requiring attention before full public launch.

  • 1 Waitlist gate active — All features are built but inaccessible to the public. The homepage is a waitlist signup form. Removing the gate exposes the full app.
  • 2 Mock session data — sessions-data.ts contains 20+ sessions with empty audioUrl fields. These are fallback data and should be replaced with real Supabase content before launch.
  • 3 Watchlist dual implementation — The watchlist page reads from Supabase, but the session player's heart/save toggle writes to localStorage. These need to be unified to Supabase.
  • 4 Downloads dual implementation — Same pattern as watchlist. The downloads page reads Supabase, the library module writes localStorage. No actual file download capability exists.
  • 5 Live room chat is mock data — The chat panel in live rooms displays hardcoded messages. It is not connected to any real-time messaging backend.
  • 6 Festivals page placeholders — 3 of 4 team members on the festivals page have placeholder text: [Role] and [Description].
  • 7 About page placeholder text — The about page contains placeholder copy that needs to be replaced with actual content.
  • 8 Rooms localStorage fallback — Group meditation rooms fall back to localStorage instead of using Supabase Realtime for participant state in certain edge cases.
  • 9 Admin auth check fragile — requireAdmin() only checks against a single email address, while the middleware supports comma-separated admin emails. These should be aligned.
